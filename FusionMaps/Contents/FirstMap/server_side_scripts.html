Creating maps using various server side scripts

So far in this section, we have shown you how to build maps using static data (that is, data which was manually created by you). However, FusionMaps XT can also connect to dynamic sources of data using server side scripts like ASP, ASP.NET C#, ASP.NET VB.NET, PHP, J2EE, Ruby on Rails, ColdFusion, etc.

FusionMaps XT can be implemented using ANY server-side technology, since maps are rendered at client side. The role of server-side scripts is only to provide XML/JSON data as string.

To make the task of embedding maps in Web pages easier, FusionMaps XT provides wrapper classes (FusionCharts wrapper classes) for various server-side scripts like ASP, ASP.NET C#, ASP.NET VB.NET, PHP, J2EE, Ruby on Rails, and ColdFusion. Moreover, in ASP and PHP, the APIs allow you to connect to arrays and databases. You can even dynamically build XML data which can directly be provided to the map.

FusionCharts wrapper classes for ASP.NET C#, ASP.NET VB, PHP and Classic ASP are present in Download Pack > Code folder. All the wrapper classes are present in FusionCharts XT pack. If you do not have the licensed pack of FusionCharts XT, you can directly download the trial version of FusionCharts XT and get the fully functional wrapper classes from the pack. The wrapper classes are present in the FusionCharts XT Download Pack's Code folder.

In this page, we provide highlights of code snippets that are used in various server-side technologies for embedding FusionMaps XT. The next section, shows how maps can be generated using dynamic data from database.

Using Classic ASP 
Call renderChart("../Maps/FCMap_World.swf", "Data.xml", "", "map1", 750, 400, False, True)

Code to create JavaScript map:

 
FC_SetRenderer( "javascript" )
Call renderChart("../Maps/FCMap_World.swf", "Data.xml", "", "map1", 750, 400, False, True)

Using ASP.NET C# 
FusionCharts.RenderChart("../Maps/FCMap_World.swf", "Data.xml", "", "map1", "750", "400", false, true);

Code to create JavaScript map:

 
FusionCharts.SetRenderer("javascript");
Literal1.Text = FusionCharts.RenderChart("../Maps/FCMap_World.swf", "Data.xml", "", "map1", "750", "400", false, true);

Using J2EE 
<fc:render mapId="map1" swfFilename="../Maps/FCMap_World.swf" 
    width="750" height="400" debugMode="false" 
     xmlUrl="Data.xml" />

Code to create JavaScript map:

 
<fc:render mapId="map1" swfFilename="../Maps/FCMap_World.swf" 
    width="750" height="400" debugMode="false" 
     xmlUrl="Data.xml" renderer="javascript"/>

Apart from embedding map in web applications, some of the server-side-scripts also help in exporting the map into image or a PDF. You can know more on this from Exporting as Image/PDF > Server Side Exporting section. For scripts that do not support server side exporting, you can always use the client side Export handlers.
